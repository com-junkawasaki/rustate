RStateとは

RStateはRustで実装された状態機械（Statechart）ライブラリです。状態管理を型安全に行い、WebAssemblyと連携することでブラウザ上でも動作します。

基本的な使い方

1. 状態定義


// Rustでの状態定義例
#[derive(Clone, Debug)]
pub enum TrafficLightState {
    Red,
    Yellow,
    Green,
}

2. イベント定義


// イベント定義例
#[derive(Clone, Debug)]
pub enum TrafficLightEvent {
    Next,
    // その他のイベント
}

3. 状態機械の実装


// 状態機械実装例
impl Statechart for TrafficLight {
    type State = TrafficLightState;
    type Event = TrafficLightEvent;
    type Context = ();
    
    fn transition(&self, state: &Self::State, event: &Self::Event, _ctx: &Self::Context) -> Option {
        match (state, event) {
            (TrafficLightState::Red, TrafficLightEvent::Next) => Some(TrafficLightState::Green),
            (TrafficLightState::Green, TrafficLightEvent::Next) => Some(TrafficLightState::Yellow),
            (TrafficLightState::Yellow, TrafficLightEvent::Next) => Some(TrafficLightState::Red),
            _ => None,
        }
    }
}

WebAssemblyとの連携

RStateはWebAssemblyとシームレスに連携できます。これにより、Rustで記述した状態機械をJavaScriptから利用できます。

Rustコード（wasm-bindgen対応）


// wasm-bindgenを使用した公開例
#[wasm_bindgen]
impl TrafficLight {
    #[wasm_bindgen(constructor)]
    pub fn new() -> Self {
        Self::default()
    }
    
    #[wasm_bindgen]
    pub fn get_state(&self) -> String {
        format!("{:?}", self.state)
    }
    
    #[wasm_bindgen]
    pub fn send(&mut self, event: &str) -> bool {
        // イベントの変換処理
        let event = match event {
            "next" => TrafficLightEvent::Next,
            _ => return false,
        };
        
        // 状態遷移の実行
        if let Some(new_state) = self.transition(&self.state, &event, &()) {
            self.state = new_state;
            true
        } else {
            false
        }
    }
}

JavaScript側の使用例


// JavaScriptからの呼び出し例
import init, { TrafficLight } from './pkg/rstate_wasm.js';

async function run() {
    await init();
    
    const trafficLight = new TrafficLight();
    console.log(trafficLight.get_state()); // "Red"
    
    trafficLight.send("next");
    console.log(trafficLight.get_state()); // "Green"
}

run();

デモ実装の解説

このサイトで提供している各デモの詳細な実装解説です。

信号機デモ

信号機デモは、基本的な状態遷移を視覚化したものです。「Red」→「Green」→「Yellow」→「Red」のサイクルを実装しています。

音楽プレーヤーデモ

音楽プレーヤーデモは、より複雑な状態管理を示しています。「Playing」「Paused」「Stopped」の状態と、再生、一時停止、停止、次の曲、前の曲などの様々なイベントを処理します。

高度な機能

ネストした状態

RStateは階層的な状態（ネストした状態）をサポートしています。これにより、複雑な状態機械を構造化して実装できます。

ガード条件

状態遷移にガード条件を追加することで、特定の条件下でのみ遷移が発生するようにできます。

副作用（アクション）

状態遷移時に副作用を発生させることができます。これにより、状態変化に応じた処理を実行できます。
